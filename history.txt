Yang udah dipelajari dari SVC (jadi dasar kompatibilitas)

Dua jalur transport yang wajib dipertahankan persis:

1. Plugin channel (TCP, Bukkit Messenger) — handshake & state. voicechat:request_secret (incoming, bawa int compatibilityVersion) → server balas voicechat:secret (16-byte Secret AES-128-GCM + port + UUID + codec + mtu + distance + keepalive + groupsEnabled + voiceHost + allowRecording). Plus paket state/grup.

2. Raw UDP (default port 24454) — audio + auth. Wire: [0xFF magic][UUID player][byte[] encryptedPayload], payload = IV(12) + AES-128-GCM ciphertext. 10 tipe paket terdaftar: MicPacket(0x1), PlayerSound(0x2), GroupSound(0x3), LocationSound(0x4), Authenticate(0x5), AuthenticateAck(0x6), Ping(0x7), KeepAlive(0x8), ConnectionCheck(0x9), ConnectionCheckAck(0xA). Audio = Opus 48kHz/20ms/960 sample.

Handshake: request secret → balas secret → client kirim AuthenticatePacket(uuid,secret) via UDP → server cek → unCheckedConnections + balas AuthenticateAckPacket → client ConnectionCheckPacket → pindah connections + balas ack. Keepalive tiap keepalive ms, timeout keepalive*10.

Hal jujur yang perlu kamu tahu

Batasan server-side murni (tanpa mod client tambahan): server bisa gate audibility, modulasi field distance (redam/mendam), noise cancellation pada PCM, dan range dinamis. Tapi reverb/low-pass "asli" di mixer client OpenAL SVC itu fixed — jadi "bergema/mendam beneran" cuma bisa dengan Tier 2 opt-in: decode → DSP bake (low-pass + convolution reverb) → re-encode. Itu mahal (latency + CPU), makai aku buat opt-in per-world, bukan default. Default = Tier 1 raytrace occlusion + modulasi distance (murah, terbukti efektif).

"Kompatibel semua versi client" = goal aspirasional. Mekanisme riilnya = negosiasi compatibilityVersion per koneksi + adapter per versi di VersionedPacketRegistry. Default jalan di versi terbaru (compat 20) dulu, adapter legacy menyusul bertahap — aku tulis ini jelas di plan, bukan janji muluk.
